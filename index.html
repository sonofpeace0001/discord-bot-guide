03
Discord app

Register your bot on the Discord Developer Portal.

This is where Discord gives your bot its identity (a token) and an invite link so it can join a server.

1
Go to discord.com/developers/applications and click New Application. Name it.
2
Open the Bot tab and click Add Bot. Copy the Token — keep it secret, you'll need it next.
3
Under Privileged Gateway Intents, enable Message Content Intent (and Server Members if your bot needs it).
4
Open the OAuth2 → URL Generator, pick scopes (bot, applications.commands) and the permissions your bot needs, then open the generated URL to invite the bot to your server.
Security: Never commit your token to GitHub. You'll pass it to Railway as an environment variable instead.
04
GitHub

Push your bot to a GitHub repository.

Railway will deploy directly from this repo, so get your code up here first.

1
Create a new repo on github.com/new (name it, e.g. my-discord-bot).
2
Put the files CREAO generated into a folder on your computer.
3
Make sure .gitignore includes .env and node_modules/ so secrets never get committed.
4
Commit and push, or just drag-and-drop the files into GitHub's web uploader.
# from your bot folder
git init
git add .
git commit -m "Discord bot from CREAO"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/my-discord-bot.git
git push -u origin main
Prefer no terminal? Use GitHub Desktop, or upload files directly on the GitHub website. The terminal commands above are optional.
05
Railway

Deploy live on Railway.

Railway reads your GitHub repo, installs dependencies, and runs your bot 24/7 in the cloud. You only configure it once.

1
Go to railway.app, sign in with GitHub, and click New Project → Deploy from GitHub repo.
2
Select the repo you just pushed.
3
Open the project Variables tab and add DISCORD_TOKEN with the token from Step 03.
4
Railway auto-detects Node.js from package.json and runs your start script. Check the Deployments tab — when it says Active, your bot is online.
Your bot needs a start command. Make sure package.json has "start": "node index.js" in its scripts. The CREAO prompt below already includes this.
# Example package.json scripts the prompt generates
"scripts": {
  "start": "node index.js"
}
06
Verify

Test it and iterate.

Your bot is live. Now confirm it works, then keep building by chatting with CREAO.

1
Go to your Discord server and type your bot's command (e.g. !ping or a slash command). It should respond.
2
Check Railway Logs if nothing happens — the token or intents are the usual culprits.
3
Want a new feature? Ask CREAO in the same chat. It updates the files — push the changes to GitHub and Railway auto-redeploys.

That's the loop: chat with CREAO → push to GitHub → Railway ships it. Repeat forever.

CREAO · Discord Bot Builder Prompt 
Build me a complete, production-ready Discord bot using discord.js (Node.js, latest v14). I will deploy it on Railway from a GitHub repo, so it must be deploy-ready.

== BOT SPEC ==
- Bot name: [My Bot Name]
- Prefix for text commands: [!]
- What the bot does: [describe what your bot should do — e.g. a moderation bot with kick/ban/warn commands, a welcome bot that greets new members, a FAQ bot, a leveling bot, etc.]
- Features I want:
  1. [feature one — e.g. !ping replies with latency]
  2. [feature two — e.g. slash command /hello that says hello to the user]
  3. [feature three — e.g. welcome message in a system channel when a member joins]
  4. [add more or remove as needed]

== TECHNICAL REQUIREMENTS ==
- Use discord.js v14 and Node.js 18+.
- Implement BOTH classic prefix commands AND slash commands (register slash commands on the ready event).
- Read the token from process.env.DISCORD_TOKEN (never hardcode it).
- Enable required intents: Guilds, GuildMessages, MessageContent, GuildMembers.
- Structure the project cleanly:
    /index.js          (entry point — logs in, registers slash commands, listens to events)
    /package.json      (with "start": "node index.js", discord.js as dependency, "type": "module" or commonjs — your call, just make it consistent)
    /.env.example      (lists DISCORD_TOKEN and any other vars, no real values)
    /.gitignore        (ignores node_modules/ and .env)
    /README.md        (setup steps: install deps, add token to .env locally, invite bot, run npm start, and how to deploy on Railway)
- Add clear comments explaining each section so I can learn and extend it.
- Include basic error handling so the bot stays online (try/catch around command execution, handle DiscordAPIError).

== DELIVERABLES ==
Give me each file with its filename as a heading, fully written and ready to save. Then give me a short "How to run locally" section and a "How to deploy to Railway" section. Keep everything copy-paste ready.

FAQ

Questions before you start?

Do I need to know how to code? +
No. The Super Agent writes the code from your description. You copy the files to GitHub and connect Railway — both have web interfaces. The terminal commands are optional shortcuts.
Is CREAO free? +
You can start free and build your bot in one session. Paid plans unlock more runs and heavier usage. Sign up through agent.creao.ai/@Sonofpeace.
Is Railway free? +
Railway offers a free trial/hobby credit to start. A small always-on Discord bot typically costs a couple of dollars per month. You can pause the deployment any time.
What if my bot doesn't respond? +
The most common causes: the DISCORD_TOKEN variable isn't set in Railway, the Message Content Intent isn't enabled in the Discord Developer Portal, or the bot lacks permissions in the server. Check Railway's Logs tab for the exact error.
Can I add more features later? +
Yes. Keep chatting with the Super Agent in CREAO — it remembers your project and updates the files. Push the new code to GitHub and Railway redeploys automatically.
Will my token stay safe? +
Yes, if you follow the guide. The token lives only in Railway's environment variables and your local .env (which is gitignored). Never paste it into chat or commit it.
